# 豆瓣爬虫技术路线说明文档

## 项目概述

本项目是一个基于 Playwright 的豆瓣关键词搜索爬虫，用于爬取豆瓣平台上包含特定关键词的小组帖子及其评论数据。项目提供 GUI 图形界面，支持用户交互式操作。

## 技术选型

### 核心技术栈

| 技术 | 版本要求 | 用途 |
|------|----------|------|
| Python | 3.10+ | 主要编程语言 |
| Playwright | 最新版 | 浏览器自动化框架 |
| BeautifulSoup4 | 最新版 | HTML 解析 |
| lxml | 最新版 | XML/HTML 解析后端 |
| tkinter | 内置 | GUI 界面 |

### 为什么选择 Playwright？

1. **真实浏览器环境** - 使用 Chromium 内核，模拟真实用户行为
2. **自动处理 JavaScript** - 豆瓣搜索页面使用 React 动态渲染
3. **支持用户交互** - 允许用户手动登录、处理验证码
4. **强大的选择器** - CSS、XPath、Text 多种选择器支持
5. **反检测能力** - 相比 requests + selenium 更难被识别

## 关键技术点

### 1. 链接解析

豆瓣搜索结果中的帖子链接采用特殊格式：

```
https://www.douban.com/doubanapp/dispatch?uri=/group/topic/344185043?_spm_id=xxx
```

需要从 `uri` 参数中提取真实的帖子路径，然后构建完整的 URL。

### 2. 验证码检测

采用多种方式检测验证码：
- 检查特定 CSS 选择器 (`.captcha`, `#captcha` 等)
- 检查 URL 是否包含 captcha/verify
- 避免使用关键词匹配（误报率高）

### 3. 防封禁策略

- **随机延迟**: 每次请求间隔 3-6 秒
- **真实浏览器**: 使用 Playwright 模拟真实用户
- **手动登录**: 不存储 Cookie，由用户手动登录
- **验证码处理**: 检测到验证码时暂停等待用户处理

### 4. 日志系统

- 实时日志显示在 GUI 界面
- 同时保存到本地文件 (`logs/crawler_YYYYMMDD_HHMMSS.log`)
- 调试页面 HTML 保存到 `logs/debug_page.html`

## 与 MediaCrawler 的关系

本项目数据格式与 MediaCrawler 保持一致：
- 数据保存在 `MediaCrawler/data/douban/json/` 目录
- 字段命名与 MediaCrawler 的其他平台（小红书、微博等）统一
- 便于后续统一处理和分析
